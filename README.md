
📌 项目概述
本系统是基于 C语言 开发的物联网设备巡检管理平台，采用模块化架构和链表数据结构，实现对物联网设备的全生命周期管理。
系统分为管理员和工程师双角色，支持设备状态监控、维修记录追踪、人员在线状态管理等功能，数据通过文件持久化技术保障安全，适用于嵌入式设备运维场景。

💡 设计哲学：
"分离关注点" —— 将菜单逻辑、数据操作、文件存储解耦，降低维护成本
"内存即安全" —— 链表动态管理内存，避免静态数组溢出风险

🌟 核心功能
| 角色 | 功能 | 技术亮点 |
|------|------|----------|
| 管理员 | • 工程师全生命周期管理（增删改查）<br>• 设备信息维护<br>• 在线状态监控 | 链表遍历优化（时间复杂度O(n)）<br>重复性校验（账号/编号唯一性） |
| 工程师 | • 设备状态查看（待维修/维修中/正常/损坏）<br>• 设备检查维修<br>• 密码修改<br>• 在线状态管理 | 状态机设计（维修流程控制）<br>维修记录历史追踪（5条记录） |

💾 数据持久化
双文件存储机制：
Engineers：存储工程师账户信息（含在线状态）
Devises：存储设备状态及维修历史

🔍 设备状态管理
四态模型：未知 → 正在检查 → 损坏/正常 → 正在维修 → 正常
智能维修记录：
自动覆盖最旧记录（当历史记录满5条时）
时间+地点双维度记录

操作指南
首次启动：自动创建空数据文件（Engineers/Devises）
管理员登录：
账号：admin
密码：123456
工程师操作：
首次使用需注册账号
登录后选择设备进行检查/维修
安全退出：
工程师可选择保留/清除"在线"状态
所有数据自动保存到文件
